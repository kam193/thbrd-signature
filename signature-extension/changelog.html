Welcome in the new version of Signature sync for Gmail!

This version contains a few significant changes. See the changelog below. Changelog is shown only on first use and major changes, but you can always access it in add-on's preferences.

Version 3.1.1

  • Additional hardening by using better random generator in OAuth flow.

Version 3.1.0

  • Fixing trying to sync already removed from Thunderbird accounts or identities.

Version 3.0.0

  • Multiple identities and Gmail send-as aliases are now supported. By default, only default identity is synced with default Gmail address. If you have additional identities, you can configure syncing signatures for them in add-on's preferences.

Version 2.1.0

  • Moving into official identity API in Thunderbid, as so the extension doesn't need the full permissions anymore. Requires Thunderbird 91 or newer.

Version 2.0.0

  • Important change. The add-on (version from Thunderbid add-on portal) contains now default OAuth credentials, but it still during certification and can handle just limited number of users.
  • Breaking change! Use "desktop app" oauth client type. Your OAuth credentials requires update. Please follow instructions.
  • Fix problem with sync stopped shortly after connection.
  • Show error popup after every 3 failed sync tries.
  • Show changelog page on major updates.
  • Dark theme support

Version 1.0.x

  • Sync signatures for selected accounts from Gmail
  • Use your own OAuth app (web client type)

Signature sync for Gmail helps you with syncing your signatures. You can report issues on Github repo.
